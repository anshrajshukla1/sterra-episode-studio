# KNOWN_ISSUES.md

Honest list of known limitations, broken things, and unfinished work.

> The assignment spec: "We trust a candidate who tells us where the bodies are buried far more than one who pretends there are none. Hiding a known bug is worse than having it."

---

## Known Limitations

### In-process job queue doesn't survive server restarts

**Severity:** Medium  
**Impact:** If the Render backend restarts (deploy, OOM, crash), any queued or running jobs are lost. The job is marked as `'failed'` in the DB on startup detection is not implemented — the job will remain in `'running'` state forever.

**Workaround:** Manually check jobs that are stuck in `'running'` state and re-trigger them.

**Fix path:** Migrate to Celery + Redis. The `worker.py` interface is designed to be swappable — `run_pipeline` is passed as a dependency, so wrapping it in a Celery task would be straightforward.

---

### No login UI

**Severity:** Low (internal tool)  
**Impact:** There is no sign-in page in the frontend. Auth is fully wired on the backend (Firebase JWT verification) and in the frontend API client (token attachment), but the UI doesn't provide a way to sign in.

**Workaround:** Use the Firebase console to create a user, then use the Firebase REST API to exchange credentials for an ID token, or temporarily disable auth middleware for local development.

**Fix path:** Add a simple `/login` page with `signInWithEmailAndPassword` (30 minutes of work, deliberately deferred to prioritize core pipeline).

---

### Video seeking requires full download

**Severity:** Low  
**Impact:** The exported video is served via `FileResponse` without byte-range support. This means the browser must download the entire video before seeking works on large exports.

**Fix path:** Replace `FileResponse` with a streaming response that respects `Range` headers.

---

### SDK hand tracking may not install cleanly

**Severity:** Low  
**Impact:** The `HandTracker` model from `stera-sdk` uses MediaPipe under the hood. On some platforms/Python versions, MediaPipe's native extensions may fail to install or load. The pipeline handles this gracefully (hand tracking is skipped with a warning) — but hand pose data won't be in the export.

**Decision:** This is explicitly handled in `pipeline.py` — the failure is logged as a warning, not an error. Hand tracking is optional per the spec.

---

### No upload UI (by design, but worth noting)

**Severity:** Low (by design)  
**Impact:** Users cannot upload recordings through the web UI. Files must be placed in `RECORDINGS_DIR` directly.

**Rationale:** See DECISIONS.md §5.5. The naive upload approach has sharp edges for multi-hundred-MB files. The configured-directory approach is more reliable.

---

### QC report iframe has CORS limitations

**Severity:** Low  
**Impact:** The QC HTML report generated by `Evaluate(session)` may reference local resources (CSS, JS) using relative paths. When embedded in an iframe served from a different origin, some resources may fail to load.

**Workaround:** The "Open in new tab" button works correctly. The iframe is a convenience feature.

---

### No pagination on recordings list

**Severity:** Low (internal tool, few recordings)  
**Impact:** All recordings are scanned and returned in a single API call. With hundreds of recordings, this could be slow.

**Fix path:** Add `limit`/`offset` query params to `GET /api/recordings` and implement cursor-based pagination in the UI.

---

### Token passed as query param for video/report

**Severity:** Low  
**Impact:** Firebase ID tokens are passed as `?token=<jwt>` for video `<src>` and report iframe `<src>` URLs, since browsers can't set `Authorization` headers on these elements. Tokens in URLs may appear in server logs.

**Mitigation:** Tokens expire in 1 hour. The API validates expiry. Server logs should be treated as sensitive.

**Fix path:** Use signed, short-lived URL tokens generated by the backend (similar to S3 presigned URLs) specifically for media access.
